Terminal

Terminal używamy w czasie pracy z naszym projektem. Umiejętność jego obsługi jest kluczowa w pracy programistycznej i obecnie jest to standard.

Ważne komendy - MacOS, Linux oraz Windows

  • cd ~ (Windows cd C:\) przejście do katalogu domowego (w Windows dysk C)
  • ls (Windows dir) wypisanie zawartości katalogu
  • cd .. przejście do katalogu wyżej
  • cd nazwa-katalogu przejście do katalogu o nazwie nazwa-katalogu
  • code . uruchomienie visual studio code w aktualnym katalogu
  • pwd czyli print working directory> (Windows cd) wyświetlenie aktualnej ścieżki
  • mkdir nazwa-katalogu stworzenie katalogu o nazwie nazwa-katalogu
  • rmdir / rm usuwa pusty katalog/ usuwa katalog nawet jak są pliki
  • --help każdy program ma opcję --help lub -h, która wyświetla krótką pomoc
  • clear (Windows cls) komenda czyszcząca ekran terminala i przesuwająca wiersz poleceń na górę ekranu

Więcej komend dla: Windows oraz Linux/MacOS

  • katalog roboczy - working directory
  • przechowalnia - stage area
  • repozytorium - .git folder

Ważne komendy - Git (ogólne)

  • git informacje o git
  • git --version sprawdzenie wersji gita
  • git init tworzymy repozytorium w danym katalogu
  • git config --global user.name "Jan Nowak" przypisanie danych użytkownika
  • git config --global user.email jannowak@example.com przypisanie meila
  • git config --global core.editor + np."code --wait" przypisujemy program do edycji komentarzy
  • git log - szczegółowa historia gita
  • git log --oneline bardziej uproszczona/ skondensowana historia gita
  • git log --author="np.Kamil" wyszukiwanie poprzez wpisanie autora
  • git log --grep wyszukiwanie poprzez wpisanie konkretnej frazy z komentarzy commita
  • git log --summary wykazuje podsumowanie plików
  • git log --stat wykazuje statystyki plików
  • git log --patch sprawdzamy co zostało konkretnie zmienione np. git log --patch -- index.html
  • git log -np.3 decydujemy ile ostatnich historii chcemy zobaczyć
  • git log --format="" oficjalna dokumentacja
  • git shortlog prosty podział z uwzględnieniem użytkowników
  • git reflog sprawdzamy historie resetów

Ważne komendy - Git (lokalnie)

  • git status sprawdzenie stanu katalogu
  • git add . dodanie wszystkich plików
  • git add + nazwa pliku dodanie konkretnego pliku
  • git commit -m "opis zmian" commitowanie
  • git clean usuwanie plików które nie zostały dodane do indexu Gita
  • git reset przenoszenie plików ze stage do working directory. odwrotność git add
  • git revert operacja polegająca na odwróceniu zmian z wybranego commitu i zapisaniu ich jako nowy commit
  • git rm / git mv usuwanie/przenoszenie plików w repozytorium
  • git checkout (path) przywracanie stanu pliku z indeksu gita. przywrócenie wprowadzonych zmian. działa tylko gdy jest wprowadzone do indexu
  • git restore(aktualnie 2020) + nazwa pliku lub . usuwa/cofa zmiany wprowadzone w katalogu roboczym (working directory)
  • git restore --staged(aktualnie 2020) + nazwa pliku lub . cofa git add do poziomu katalogu roboczego
  • git stash Dodaje zmiany na stos
  • git stash push j/w. ale umożliwia dodanie flagi -m "[komentarz]"
  • git stash show (-p —patch) pokazuje jakie zmiany znajdują się na stosie
  • git stash list wyświetla listę zmian odłożonych na stos
  • git stash pop (apply & drop) przywraca zmiany ze stosu (i usuwa je z niego)
  • git stash drop usuwa zmiany ze stosu
  • git stash apply przywraca zmiany ze stosu (zostawiając je jednocześnie)
  • git stash branch tworzy nowy branch zawierający zmiany z wybranego stosu

Ważne komendy - Git (zdalnie)

  • git push przesłanie zapisanego commitu(zmian lub nowego pliku) do zdalnego repozytorium
  • git fetch - pobieramy zmiany ze zdalnego repozytorium
  • git merge + origin/master - łączymy zmiany lokalnego repozytorium ze zdalnym
  • git remote + np show -zarządzamy repozytoriami + wykaz informacji o repozytorium
  • git pull (jest to połączenie git fetch i git merge - łączenie, przesyłanie danych z zdalnego repozytorium do lokalnego.
  • git clone po utworzeniu repozytorium w githubie(zdalnym repozytorium) w terminalu klonujemy repozytorium czyli łączymy githuba(zdalne repozytorium) z gitem. Ważne! dodajemy clona w folderze dev
  • git rebase - sprawia ze historia comitów jest bardziej czytelna
  • git tag - wykaz wszystkich etykiet
  • git tag -a - tworzenie etykiet w commitach

Branch - podstawy

  • git branch - sprawdzamy na jakim branchu jesteśmy. Podstawowy to master
  • git branch nazwa nowego brancha - tworzymy nowy branch
  • git branch -D + nazwa brancha -usuwamy branch
  • git checkout nazwa brancha - przechodzimy na wybrany branch
  • git push + formułka z gita (kopiuj/wklej) - wypychamy nasz branch do repozytorium na git hubie
  • git merge nazwa brancha który chcemy dodać - łączymy nasz nowy branch z głównym branchem master. Komende wpisujemy będąc na branchu, do którego chcemy dołączyć nowy branch.
  • git log --graph --decorate --all --oneline pełny wykaz branchy

Przykładowe branche

  • Master branch - główny branch (publikowany)
  • Dev branch - testowy branch (alfa/beta)
  • Feature branch - branch wprowadzający nowe funkcje
  • User branch -prace indywidualne pojedyńczego członka grupy
  • Test/ Bugfix branch - branch do testów, wykrywanie błędów

WTF Webpack - instalacja oraz instrukcja obsługi

  1. Utwórz z szablonu nowe repozytorium i sklonuj do siebie na dysk - wtf webpack
  2. npm install - instalujemy npm do naszego projektu
  3. npm run start -odpalamy webpack w naszym projekcie.
  4. http://localhost:8080/ - mam dostęp do lokalnego serwera bez konieczności używania wtyczek z Visual Studio Code
  5. npm run build - do zmiany trybu build
  6. ../ - dodajemy obrazki używając podpowiedzi z VSC
  7. webpack.common.js - w tym pliku dodajemy dodatkowe pliki html-owe
  8. chunks - sa to tablice do których możemy przypiąć indywidualnie pliki js
  9. import "../scss/main.scss"; - ta linijka kodu importuje nam style - nie usuwamy jej!!!
  10. ~ - używamy tyldy w sasse do importowania plików z node_modules
  11. source map - jest to ściaga w devtool
  12. DIST - jest wynikiem naszej pracy, zawartość tego katalogu publikujemy na serwerze
  13. npm run publish - generujemy w git hubie zakładkę gh-pages branch - publikujemy tym strone

  • NPM - menadżer paczek do node.
    • jest to repozytorium, z którego możemy pobierać przydatne paczki do naszego projektu
    • dodajemy do projektów sporo gotowych bibliotek/narzędzi
    • nie można polegać na konfiguracji edytora/jego wtyczkach
    • universalne źródło ze wszystkimi narzedziami
    • dostep do aktualizacji
    • npm init dodaje npm do projekty(robimy to tylko raz). W projekcie tworzy nam się plik package.json
    • npm init -y dodaje npm do projekty z domyślnymi danymi wygenerowanymi przez program
    • npm + install + nazwa paczki + --save instalujemy paczkę. W części dependencies wpisuje się nam paczka. Tworzy się dodatkowy plik package-lock.json, który blokuje wersje oraz folder z plikami node_modules. Możemy dodać go do repozytorium.
    • npm + install + nazwa paczki + --save-dev instalowane paczki w przypadku gdy dotyczą tylko projektu (wewnątrz Visual Studio Code) a nie strony
    • npm install instaluje /szuka paczek
    • nie commitujemy node_moduls - ukrywamy je za pomocą .gitignore
