Co ten JAVASCRIPT?

Informacje o JS

"JavaScript != Java"

  • Będziemy uczyć się ECMAScript 2015+ (ES6+)
  • ECMAScript jest to ustandaryzowany przez organizację ECMA skryptowy język programowania
  • JavaScript implementuje rzeczy ze specyfikacji ECMAScript
  • Rozwój ECMAScript mocno przyspieszył przez szerokie wykorzystywanie JavaScriptu
  • ES6 (i nowsze) wprowadzają wiele udogodnień dla developerów
  • Nowe feature'y mogą nie być wspierane we wszystkich przeglądarkach

Zapis JS w HTML/odrębym pliku .js

Notatki z lekcji nr.1

  • Pliki zawierające kod JavaScript mają rozszerzenie .js ale podobnie jak style - mają też swój znacznik - <script>
  • alert ('parametry') - polecenie wyskakującego okna na początku otwierania strony
  • console.log('text') - polecenie umożliwiające dodanie treści do konsoli
  • console.log(zmienna)
  • Wartości są niezmienne
  • const - Tworzy stałą, która może być globalna lub lokalna dla funkcji, która ją zadeklarowała. Zasady zasięgu dla stałych są takie same jak dla zmiennych. Wartość stałej nie może zostać zmieniona poprzez ponowne przypisanie; stała nie może także być ponownie zadeklarowana. Stała nie może mieć takiej samej nazwy jak funkcja lub zmienna o tym samym zasięgu.
  • backtick/backquote `` - specjalny znacznik/literał, w którym możemy opakować tekst i zmienne
  • ${zmienna} - do opakowania zmiennych

Podsumowanie lekcji pierwszej

  • Kod JS uruchamia się raz (po załadowaniu strony)
  • Java to nie JavaScript
  • JavaScript implementuje rzeczy ze specyfikacji ECMAScript
  • JavaScript to frontendowe programowanie
  • Za pomocą JS można robić różne szalone rzeczy

Notatki z lekcji nr.2

  • Zmienna to takie połączenie z naszym wszechświatem wartości
  • Zmienna to nie wartość - zmienna wskazuje położenie naszej wartości
  • let - pierwszy sposób zapisu definicji zmiennych
  • const - drugi sposób zapisu definicji zmiennych
  • var - trzeci sposób zapisu definicji zmiennych(powoli wychodzący z obiegu)
  • Zmienne w JS są jak młody pelikan: "łykną" wszystko
  • typeof - sprawdzanie typu zmiennych
  • IMMUTABILITY - Nie możemy zmieniać wartości primitive values w JS. To co zmieniamy to połączenie między nimi, a zmiennymi.

Typy proste w JS (primitives)

  • String - tekst, otaczamy ' lub " (zalecam ')
  • Number - liczba
  • Bigint - duża liczba ;) większa niż 253
  • boolean - true lub false
  • null - wartość celowo niezdefiniowana
  • undefined - brak definicji
  • *symbol - tworzy "unikalne opakowanie"

Notatki z lekcji nr.3

  • DOM - Document Object Model
  • document.querySelector('.selector') - wykaz tresci(węzeł) z html w consoli
  • block__element--js - używamy tej dodatkowej klasy gdy chcemy użyć querySelector w js
  • nazwaZmiennejQuerySelector.innerHTML = 'treść' - umożliwia wprowadzenie zmian/dodanie nowej wartości z js do html
  • console.log(nazwazmiennej.innerHTML) - sprawdzamy treść elementu

Podsumowanie lekcji trzeciej

  • DOM to "złożony" w przeglądarce HTML
  • JavaScript może modyfikować DOM
  • funkcja querySelector zwraca node (węzeł) z HTML-a, selektor jest zapisywany zgodnie z css
  • korzystając z BEM - tworzymy dodatkową klasę z modyfikatorem --js (ułatwia identyfikację / usuwa potencjalne problemy) DOM

FUNKCJE

  • return - definiuje co ma być zwrócone
  • Deklarujemy nazwę funkcji (nie zawsze)
  • W środku dokonujemy obliczeń / wykonujemy zadania
  • Funkcja przyjmuje argumenty (ale nie musi)
  • Funkcja zwraca zawsze coś (nawet jeśli nie mamy return - wtedy funkcja zwraca undefined)

Funkcje po co?

  • Deklarujemy kod, który się powtarza (DRY - don't repeat yourself)
  • Zwiększamy czytelność kodu
  • Łatwiejsze szukanie błędów
  • Odpowiedzialność za wykonywanie zadania w jednym miejscu
  • (Fat) arrow function:
    • Skraca zapis deklaracji funkcji
    • Zwiększa czytelność (dyskusyjne)
    • Nie tworzy nowego kontekstu this, nie ma dostępu do arguments i super
    • Nie jest zamiennikiem "tradycyjnej" funkcji (ale w większości przypadków nie zobaczysz różnicy)

            KOERCJA typów to inaczej ZMIANA typów

            1. 1 == 1            => return 1 === 1
            2. null == undefined  => return true
            3. undefined == null  => return true
            4. 1 == '1'           => return 1 == Number('1')
            5. '1' == 1           => return Number('1') == 1
            6. true == ...        => return Number(true) == ...
            7. ... == true        => return ... == Number(true)
            8. '1' == [1, 2, 3]   => return '1' == ToPrimitive([1, 2, 3])
            9. [1, 2, 3] == '1'   => return ToPrimitive([1, 2, 3]) == '1'
            10. return false
