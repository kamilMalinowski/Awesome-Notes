Co ten Terminal?

Terminal

Terminal używamy w czasie pracy z naszym projektem. Umiejętność jego obsługi jest kluczowa w pracy programistycznej i obecnie jest to standard.

Ważne komendy - MacOS

  • cd ~ przejście do katalogu domowego
  • ls wypisanie zawartości katalogu
  • cd .. przejście do katalogu wyżej
  • cd nazwa-katalogu przejście do katalogu o nazwie nazwa-katalogu
  • code . uruchomienie visual studio code w aktualnym katalogu
  • pwd wyświetlenie aktualnej ścieżki
  • mkdir nazwa-katalogu stworzenie katalogu o nazwie nazwa-katalogu

Ważne komendy - Git

  • git informacje o git
  • git --version sprawdzenie wersji gita
  • git config --global user.name "Jan Nowak" przypisanie danych użytkownika
  • git config --global user.email jannowak@example.com przypisanie meila
  • git init tworzymy repozytorium w danym katalogu
  • git status sprawdzenie stanu katalogu
  • git add . dodanie plików/mian w plikach
  • git commit -m "opis zmian" commitowanie
  • git push przesłanie zapisanego commitu do githuba
  • git clone po utworzeniu repozytorium w githubie w terminalu klonujemy repozytorium czyli łączymy githuba z gitem. Ważne! dodajemy clona w folderze dev

Git dalej

  • .gitignore - jest to dodatkowy plik, który możemy dodać do naszego projekty w Visual Studio Code. Przy commitowaniu pliki oraz treść umieszczone w tym dokumencie nie są widoczne.

Node.js

  • Node.js - jest to JavaScipt używany poza przeglądarką. Nie mamy dostępu do DOM.
  • node + nazwa pliku np. node script.js otwieramy node w terminalu
  • node -v sprawdzamy wersję node.js/prawidłową instalację

NPM

  • NPM - menadżer paczek do node.
    • jest to repozytorium, z którego możemy pobierać przydatne paczki do naszego projektu
    • dodajemy do projektów sporo gotowych bibliotek/narzędzi
    • nie można polegać na konfiguracji edytora/jego wtyczkach
    • universalne źródło ze wszystkimi narzedziami
    • dostep do aktualizacji
    • npm init dodaje npm do projekty(robimy to tylko raz). W projekcie tworzy nam się plik package.json
    • npm init -y dodaje npm do projekty z domyślnymi danymi wygenerowanymi przez program
    • npm + install + nazwa paczki + --save instalujemy paczkę. W części dependencies wpisuje się nam paczka. Tworzy się dodatkowy plik package-lock.json, który blokuje wersje oraz folder z plikami node_modules. Możemy dodać go do repozytorium.
    • npm + install + nazwa paczki + --save-dev instalowane paczki w przypadku gdy dotyczą tylko projektu (wewnątrz Visual Studio Code) a nie strony
    • npm install instaluje /szuka paczek
    • nie commitujemy node_moduls - ukrywamy je za pomocą .gitignore

NPX

  • NPX - narzędzia konsolowe używane w terminalu
    • npm cowsay + dowolny tekst - zabawna paczka z krową - odpalamy ją w terminalu. Możemy ją zapisać :)

Script

  • package.json - wewnątrz pliku znajdują się skrypty, w których możemy dodawać komendy (przykład w repozytorium node-playground
  • npm run +nazwa skryptu odpalamy skrypt w terminalu

WTF Webpack - instalacja oraz instrukcja obsługi

  1. Utwórz z szablonu nowe repozytorium i sklonuj do siebie na dysk - wtf webpack
  2. npm install - instalujemy npm do naszego projektu
  3. npm run start -odpalamy webpack w naszym projekcie.
  4. http://localhost:8080/ - mam dostęp do lokalnego serwera bez konieczności używania wtyczek z Visual Studio Code
  5. npm run build - do zmiany trybu build
  6. ../ - dodajemy obrazki używając podpowiedzi z VSC
  7. webpack.common.js - w tym pliku dodajemy dodatkowe pliki html-owe
  8. chunks - sa to tablice do których możemy przypiąć indywidualnie pliki js
  9. import "../scss/main.scss"; - ta linijka kodu importuje nam style - nie usuwamy jej!!!
  10. ~ - używamy tyldy w sasse do importowania plików z node_modules
  11. source map - jest to ściaga w devtool
  12. DIST - jest wynikiem naszej pracy, zawartość tego katalogu publikujemy na serwerze
  13. npm run publish - generujemy w git hubie zakładkę gh-pages branch - publikujemy tym strone

Branch - podstawy

  • git branch - sprawdzamy na jakim branchu jesteśmy. Podstawowoy to master
  • git branch nazwa nowego brancha - tworzymy nowy branch
  • git checkout nazwa brancha - przechodzimy na wybrany branch
  • git push + formułka z gita (kopiuj/wklej) - wypychamy nasz branch do repozytorium na git hubie
  • git merge nazwa brancha który chcemy dodać - łączymy nasz nowy branch z głównym branchem master. Komende wpisujemy będąc na branchu, do którego chcemy dołączyć nowy branch.
